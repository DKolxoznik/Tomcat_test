# Мой первый сайт на Apache Tomcat

Практическое задание: личная веб-страница на Tomcat с отработкой типичных ошибок.

---

## Что сделано

- **Страница** в папке `my-site`: заголовок с именем, краткая биография, список хобби, форма обратной связи (имя, email, сообщение) и кнопка «Отправить» (пока без отправки на сервер).
- **Сервер**: проект собирается в WAR и запускается на встроенном Tomcat через Maven (плагин `tomcat7-maven-plugin`).

---

## Как запустить в IntelliJ IDEA

### Вариант 1: Готовый запуск (рекомендуется)

1. Откройте проект **Tomcat_test** в IntelliJ IDEA.
2. В правом верхнем углу выберите конфигурацию **«Tomcat Run»**.
3. Нажмите **Run** (зелёный треугольник) или **Shift+F10**.
4. Дождитесь в консоли сообщения вида: `INFO: Starting ProtocolHandler`.
5. Откройте в браузере: **http://localhost:8081/Tomcat_test/my-site/index.html**

### Вариант 2: Через Maven

1. Откройте **Maven** (правая панель) → **Tomcat_test** → **Plugins** → **tomcat7**.
2. Дважды щёлкните **tomcat7:run**.
3. После запуска откройте в браузере: **http://localhost:8081/Tomcat_test/my-site/index.html**

### Вариант 3: Терминал в IDEA

```bash
mvn tomcat7:run
```

После запуска откройте в браузере: **http://localhost:8081/Tomcat_test/my-site/index.html**

## Отработка ошибок (по заданию)

### 1. Ошибка 404 (файл не найден)

- **Сделать:** удалите файл `src/main/webapp/my-site/index.html` (или переименуйте его).
- **Проверка:** перезапустите Tomcat (если нужно), откройте  
  http://localhost:8081/Tomcat_test/my-site/index.html  
  и нажмите F12 → вкладка **Console** (или **Network**): увидите 404.
- **Исправление:** верните `index.html` на место (или создайте заново из кода в репозитории) и обновите страницу — она снова откроется.

### 2. Ошибка в CSS

- **Сделать:** откройте `src/main/webapp/my-site/styles.css` и удалите одну закрывающую скобку `}` (например, в конце файла).
- **Проверка:** сохраните файл, обновите страницу в браузере — стили перестанут применяться. В F12 → **Console** могут быть ошибки разбора CSS.
- **Исправление:** верните закрывающую скобку `}` так, чтобы все блоки `{ ... }` были корректно закрыты, сохраните и обновите страницу — стили снова заработают.

### 3. Ошибка в конфигурации (недопустимый порт)

- **Сделать:** в `pom.xml` в секции `tomcat7-maven-plugin` → `configuration` измените порт на недопустимый, например:
  ```xml
  <port>99999</port>
  ```
- **Проверка:** запустите **Tomcat Run** (или `mvn tomcat7:run`). В консоли появится ошибка (например, о недопустимом порте или исключение при старте).
- **Исправление:** верните порт в `pom.xml` на допустимый, например:
  ```xml
  <port>8080</port>
  ```
  Сохраните и снова запустите — сервер должен стартовать.

## Структура проекта

```
Tomcat_test/
├── pom.xml                    # Maven: WAR, плагин Tomcat (порт 8081, путь /Tomcat_test)
├── README.md                  # Этот файл
├── src/main/
│   ├── java/                  # Исходники Java (для проекта не используются)
│   └── webapp/
│       ├── WEB-INF/
│       │   └── web.xml        # Приветственная страница → my-site/index.html
│       └── my-site/
│           ├── index.html     # Главная страница сайта
│           ├── styles.css    # Стили
│           └── script.js     # Обработка формы (без отправки)
└── .idea/runConfigurations/
    └── Tomcat_Run.xml         # Конфигурация запуска Tomcat в IDEA
```

Имя в заголовке страницы и текст «Обо мне» можно изменить в `my-site/index.html`.
